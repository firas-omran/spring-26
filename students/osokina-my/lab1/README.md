# Лабораторная работа 1: Бинарное решающее дерево ID3 с критерием Джини

## Теоретические основы

### Решающее дерево

Решающее дерево — модель классификации, строящаяся путём последовательного разбиения пространства признаков. В каждой внутренней вершине задаётся условие по одному признаку; по рёбрам спускаемся в зависимости от значения признака, в листовой вершине — предсказание класса.

### Алгоритм ID3

ID3 (Iterative Dichotomiser 3) строит дерево жадно:
- выбирается признак, дающий наибольший прирост информации;
- данные разбиваются по выбранному признаку;
- рекурсивно строится дерево для каждого подмножества.

### Критерий Джини

Для выборки \(U\) с метками классов критерий Джини:
$$G(U) = 1 - \sum_{k} p_k^2,$$
где \(p_k\) — доля класса \(k\) в \(U\).

При разбиении по признаку \(f\) на подмножества \(U_1, U_2\):
$$\Delta G = G(U) - \frac{|U_1|}{|U|}G(U_1) - \frac{|U_2|}{|U|}G(U_2).$$
Выбирается признак с максимальным \(\Delta G\).

### Обработка пропусков

**Обучение:**
- при расчёте Gain для признака \(f\) объекты с пропуском по \(f\) исключаются;
- сохраняются оценки \(p_{left} = |U_1|/|U|\), \(p_{right} = |U_2|/|U|\) — вероятности перехода в левое и правое поддерево;
- в листьях оценка \(P(y|x \in leaf)\) вычисляется по hold-out выборке для несмещённости.

**Классификация:**
- если значение признака определено — переход по ветви;
- если пропуск — объект «распределяется» по обеим ветвям с весами \(p_{left}\), \(p_{right}\), вероятности классов усредняются.

### Редукция дерева (Pruning)

Reduced error pruning:
- post-order обход дерева;
- если оба ребёнка — листья, пробуем заменить вершину на лист (мажоритарный класс);
- если качество на валидации не ухудшается — оставляем замену.

## Особенности реализации

1. **Датасет:** Adult (UCI) — категориальные (workclass, education, occupation, …) и количественные (age, hours_per_week, …) признаки, пропуски в workclass и occupation.

2. **Бинаризация:** категориальные → one-hot; количественные → по медиане (≤ медиана / > медиана). Пропуски сохраняются как NaN.

3. **ID3:** бинарные признаки; Gain по Gini; объекты с пропуском исключаются при расчёте Gain; hold-out 20% для оценки вероятностей в листьях.

4. **Пропуски при классификации:** объект спускается в обе ветви с весами \(p_{left}\), \(p_{right}\); итоговая вероятность — взвешенная сумма по всем достигнутым листьям.

5. **Редукция:** post-order pruning по валидационной выборке.

## Сравнение результатов

- Сравниваются метрики до и после редукции (accuracy на train/val/test).
- Сравнение с sklearn `DecisionTreeClassifier(criterion="gini")` — эталонная реализация бинарного дерева.
- sklearn не обрабатывает пропуски «встроенно», поэтому при наличии NaN их нужно заполнять или удалять; ID3 реализует вероятностную обработку пропусков.

## Выводы

- Редукция уменьшает переобучение и упрощает дерево (меньше листьев и глубина).
- Метрики на val и test после редукции улучшились.
- Наличие пропусков требует специальной обработки; вероятностная схема позволяет обучать и предсказывать без дозаполнения.
